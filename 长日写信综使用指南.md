# 📮 长日写信综 - 完整使用指南

一个独立的信件系统，支持正式信件、混乱寄信、撤回机制和赏金制度。

---

## 系统概览

### 核心功能

| 功能 | 命令 | 说明 |
|------|------|------|
| 发送信件 | `发送信件` | 正式、格式化的书信系统 |
| 寄信 | `寄信` | 混乱效果的非正式信件 |
| 撤回 | `撤回` | 取消3分钟内发送的信件 |
| 查看状态 | `信件状态` | 查看配置和个人额度 |
| 查看记录 | `信件记录` | 查看今日发送统计 |
| 设置配置 | `信件设置` | 【管理员】配置系统参数 |

### 核心特性

✨ **发送信件**
- 正式、有格式的书信系统
- 支持日期、附件、自定义署名
- 可选赏金机制（写信币）
- 每日限额可配置

🕊️ **寄信**
- 通过"鸽子"传送的非正式信件
- 内容可能被侵蚀、丢失、涂黑
- 署名可能被篡改
- 信件可能送错收件人
- 带有冷却和每日限额

↩️ **撤回**
- 3分钟内可撤回已发送消息
- 仅发送者可操作
- 超时自动清除

## 玩家命令

### 1. 发送信件（正式书信）

#### 基础用法

```
。发送信件
【收件人】小明
【内容】亲爱的小明，今天天气真好...
```

#### 完整格式

```
。发送信件
【收件人】角色名
【内容】信件内容
【日期】日期（选填）
【附件】附加内容（选填）
【署名】落款（选填，默认为你的角色名）
```

#### 参数说明

| 参数 | 是否必填 | 说明 | 示例 |
|------|---------|------|------|
| 收件人 | ✅ | 接收信件的角色名 | 小明 |
| 内容 | ✅ | 信件正文 | 你好啊，好久不见... |
| 日期 | ❌ | 显示在信件顶部 | 2026年4月28日 |
| 附件 | ❌ | 额外备注，以分隔线显示 | 随信附赠礼物一份... |
| 署名 | ❌ | 落款，默认为你的角色名 | 小红敬上 |

#### 示例

**简单信件**：
```
。发送信件
【收件人】小明
【内容】嗨！你好吗？
```

**完整信件**：
```
。发送信件
【收件人】小明
【内容】亲爱的小明，

我们已经很久没有见面了。今天我想起了我们过去一起玩耍的日子，时光真的过得很快。希望你最近一切安好，期待我们能很快再次见面。

你忠诚的朋友
【日期】2026年4月28日
【附件】随信附赠: 一张泛黄的照片，记录了我们美好的回忆
【署名】小红敬上
```

#### 投递规则

- 信件发送到**收件人的私人群**
- 收件人会收到 `@` 提示
- 如果有赏金机制，会立即发放**写信币**

#### 赏金机制

如果管理员启用了写信币赏金：
- 需要达到**最少字数**才能获得赏金
- 每封信获得**固定写信币**
- 写信币可用于其他系统

**示例**：
```
✉️ 信件已送达「小明」！
🖋️ 落款：小红敬上
📅 D123（今日剩余：4/5）
💰 写信币 +10（共 50）
```

### 2. 寄信（混乱信件）

#### 基础用法

```
。寄信 小明 你好啊！
```

#### 格式

```
。寄信 <收件人> <内容>
```

#### 参数说明

| 参数 | 说明 | 例子 |
|------|------|------|
| 收件人 | 接收信件的角色名 | 小明 |
| 内容 | 信件内容 | 你好，最近怎么样... |

#### 可能发生的"混乱事件"

根据管理员设置，以下事件**可能发生**：

| 事件 | 概率 | 效果 |
|------|------|------|
| 📝 内容侵蚀 | 可配 | 内容被特殊字符替换 |
| 🔲 内容涂黑 | 可配 | 部分内容被涂黑 |
| 📄 内容丢失 | 可配 | 信件被截断 |
| ✏️ 署名篡改 | 可配 | 落款人名被篡改 |
| 📮 错送 | 可配 | 信件被发给了其他人 |
| 📢 公开 | 可配 | 信件在公开群显示 |

#### 冷却机制

- **冷却时间**：默认1分钟（可配）
- 违反冷却会被提示：`⏳ 鸽子正在休息，请X分钟后再试`

#### 每日限额

- **默认限额**：5封/天
- 超过限额提示：`🕊️ 今日寄信次数已达上限(5)`

#### 撤回选项

- 寄出后 **3分钟内可撤回**
- 发送者收到提示：`🕊️ 信件已由鸽子衔往 小明 处。今日已发 1/5（3分钟内可发「撤回」取消）。`

#### 示例

```
。寄信 小明 嘿！你在吗？
```

✅ 成功：
```
🕊️ 信件已由鸽子衔往 小明 处。今日已发 1/5（3分钟内可发「撤回」取消）。
```

❌ 冷却中：
```
⏳ 鸽子正在休息，请 1 分钟后再试
```

### 3. 撤回

#### 用法

```
。撤回
```

#### 限制

- 仅限 **3分钟内** 发送的消息
- 仅有 **发送者** 可以撤回
- 超时自动清除

#### 示例

✅ 成功撤回：
```
↩️ 短信已撤回。
```

❌ 无可撤回消息：
```
❌ 没有可撤回的消息。
```

❌ 已超时：
```
⏰ 消息已超时（3分钟），无法撤回。
```

## 管理员命令

### 信件设置

#### 用法

```
。信件设置 <参数> <值>
```

#### 【发送信件】参数

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| 直信日限 | 每日最多可发送的正式信件数 | 5 | 1-999 |
| 写信币赏 | 每封信获得的写信币数（0为禁用） | 0 | 0-999 |
| 最小字数 | 获得赏金的最少字数（0为无限制） | 0 | 0-999 |

#### 【寄信】参数

| 参数 | 说明 | 默认值 | 范围 |
|------|------|--------|------|
| 寄信日限 | 每日最多可寄信数 | 5 | 1-999 |
| 寄信冷却 | 寄信间隔时间（分钟） | 1 | 0-60 |
| 错送率 | 信件被送错收件人的概率 | 0 | 0-100 |
| 篡改率 | 内容被篡改的概率 | 0 | 0-100 |
| 签名错 | 署名被篡改的概率 | 0 | 0-100 |
| 公开率 | 信件在公开群显示的概率 | 50 | 0-100 |

。信件设置 寄信冷却 2
✅ 已设置 寄信冷却 = 2

。信件设置 错送率 20
✅ 已设置 错送率 = 20

。信件设置 篡改率 30
✅ 已设置 篡改率 = 30

。信件设置 签名错 15

## 推荐配置方案

### 方案一：纯社交向（安全信件）

```
。信件设置 直信日限 10
。信件设置 写信币赏 0
。信件设置 寄信日限 0
```

**特点**：
- 只有发送信件功能
- 没有混乱效果
- 适合正式交流

### 方案二：轻度混乱（平衡）

```
。信件设置 直信日限 5
。信件设置 写信币赏 5
。信件设置 寄信日限 3
。信件设置 寄信冷却 1
。信件设置 错送率 10
。信件设置 篡改率 15
。信件设置 签名错 5
。信件设置 公开率 30
```

**特点**：
- 正式信件有赏金激励
- 寄信有轻度混乱效果
- 偶尔会发生趣味事件

### 方案三：高度混乱（娱乐向）

```
。信件设置 直信日限 10
。信件设置 写信币赏 0
。信件设置 寄信日限 10
。信件设置 寄信冷却 0
。信件设置 错送率 50
。信件设置 篡改率 50
。信件设置 签名错 50
。信件设置 公开率 80
```

**特点**：
- 强调娱乐和随机性
- 消息经常被篡改
- 充满惊喜和笑料

## 常见问题

### 关于发送信件

**Q: 信件会立即送达吗？**
A: 是的，信件会立即发送到收件人的私人群，并会@收件人。

**Q: 能否修改已发送的信件？**
A: 不能。可以3分钟内撤回，然后重新发送修改后的内容。

**Q: 赏金是自动发放的吗？**
A: 是的，如果达到最少字数要求，赏金会自动发放。

**Q: 写信币有什么用？**
A: 这是一个可扩展的属性，可以用于其他系统中（如商城、合成等）。

### 关于寄信

**Q: 我的信件为什么没有被送出去？**
A: 可能原因：
- 冷却时间未到（提示中会告诉你还需等待多久）
- 每日限额已达上限
- 收件人角色不存在

**Q: 信件被送错了怎么办？**
A: 这是混乱效果的一部分，可以重新寄一次。如果内容很重要，建议使用"发送信件"而非"寄信"。

**Q: 内容被篡改了，是BUG吗？**
A: 不是。这是寄信的特色功能。如果不希望信件被篡改，请使用"发送信件"命令。

**Q: 公开率是什么意思？**
A: 一部分信件可能会在公开群显示，相当于被"见证"了。

### 关于撤回

**Q: 3分钟是怎么计算的？**
A: 从发送完成时开始计算，超过3分钟后无法撤回。

**Q: 撤回后信件会如何？**
A: 管理员可以看到历史记录，但玩家不会收到任何通知。

| 发送信件限额 | `direct_letter_day_counts` | 玩家每日发送信件数 |

| 冷却记录 | `chaos_letter_cooldown_*` | 寄信冷却时间记录 |

## 最佳实践

### 对于玩家

1. **选择合适的命令**
   - 正式、重要内容 → `发送信件`
   - 非正式、娱乐内容 → `寄信`

2. **充分利用功能**
   - 使用日期、附件、署名让信件更生动
   - 定期查看"信件状态"了解额度
   - 重要内容发送前先撤回一次测试

3. **撤回机制**
   - 发送后立即检查是否满意
   - 如果不满意，3分钟内立即撤回
   - 超时无法撤回，要谨慎

### 对于管理员

1. **初期配置**
   - 建议先用方案一（安全信件）测试
   - 逐步增加混乱效果，观察玩家反应

2. **平衡维护**
   - 定期检查玩家反馈
   - 根据情况调整赏金和限额
   - 监控写信币是否过剩或短缺

3. **扩展方向**
   - 与其他系统集成（如合成、商城）
   - 创建信件相关的任务或事件
   - 统计信件使用数据进行平衡
