## 物品系统

### 概念说明

物品系统是一个通用的物品管理框架，支持各类物品的注册、使用、转售等。

**两种物品类型**：
- **普通物品**：玩家对自己使用（如恢复药）
- **互动物品**：玩家对他人使用（如治疗术、诅咒术）

### 注册普通物品

```
上载物品 物品名*描述*使用次数*属性效果*允许二手
```

**参数说明**：

| 参数 | 说明 | 示例 |
|------|------|------|
| 物品名 | 显示给玩家的名称 | 医疗包 |
| 描述 | 物品说明 | 恢复生命的医疗包 |
| 使用次数 | -1:无限 / 0:一次性 / >0:指定次数 | -1, 0, 3 |
| 属性效果 | 效果说明（可选） | HP+50 |
| 允许二手 | Y/是 或 N/否 | Y |

**使用次数说明**：
- `-1`：无限使用（不会消失）
- `0`：一次性（使用后消失）
- `>0`：指定次数（使用N次后消失）

**示例**：
```
上载物品 医疗包*恢复生命*3*HP+50*N
上载物品 魔法卷轴*一次性法术*0*MP+30*Y
上载物品 永久属性书*永久增强*-1*ATK+5*N
```

### 注册互动物品

```
上载互动物品 物品名*描述*使用次数*属性效果*允许二手
```

**参数与普通物品相同**，区别在于使用方式：

| 类型 | 使用方式 | 说明 |
|------|---------|------|
| 普通物品 | `普通使用 ITEM_001` | 对自己使用 |
| 互动物品 | `施加 INTER_001 <目标>` | 对他人使用 |

**示例**：
```
上载互动物品 治疗术*为他人治疗*5*HP+50*N
上载互动物品 诅咒术*对他人诅咒*1*理智-30*N
```

### 物品操作

**使用普通物品**：
```
普通使用 ITEM_001
```
物品效果立即作用于使用者。

**使用互动物品**：
```
施加 INTER_001 李四
```
对指定玩家施加效果。

**查看互动物品配置**：
```
施加 查看 INTER_001
```

### 背包管理

**查看自己的背包**：
```
查看背包
```

**查看其他玩家的背包**（管理员）：
```
查看背包 角色名
```

**直接调整玩家背包**（管理员）：
```
调整 角色名 ITEM_001 +5   # 增加5个
调整 角色名 ITEM_001 -2   # 减少2个
```

### 物品列表查看

```
物品列表
```
显示所有已注册的物品。

## 属性系统

### 注册属性

```
我创建属性 属性名 [最小值] [最大值] [默认值]
```

**参数**：
- **属性名**：属性标识（如"体力"、"ATK"）
- **最小值**：属性下限（可选）
- **最大值**：属性上限（可选）
- **默认值**：新角色初始值（可选，默认0）

**示例**：
```
我创建属性 HP 0 1000 100
我创建属性 MP 0 500 50
我创建属性 ATK 0 100 40
我创建属性 DEF 0 50 30
我创建属性 AGI 0 50 40
```

```
设置属性 角色名 属性名 100
```

**增加/减少属性**：

## 装备系统

### 基础概念

装备系统允许玩家装备武器、防具等来增强属性。

**装备槽位**：

| 代码 | 名称 | 用途 |
|------|------|------|
| head | 头部 | 头盔、王冠、面罩等 |
| chest | 胸部 | 甲胄、衣服、外套等 |
| hand | 手部 | 武器、盾牌、手套等 |
| leg | 腿部 | 护腿、裤子、靴裤等 |
| foot | 脚部 | 靴子、鞋子等 |

### 注册装备（管理员）

```
注册装备 装备名*描述*槽位*基础属性
```

**参数**：

| 参数 | 说明 | 示例 |
|------|------|------|
| 装备名 | 显示给玩家的名称 | 铁制短剑 |
| 描述 | 装备说明 | 一把普通的短剑 |
| 槽位 | head/chest/hand/leg/foot | hand |
| 基础属性 | 装备自带的属性加成 | ATK+15,DEF+10 |

**属性格式**：
```
ATK+15              # 增加15点攻击力
DEF-5               # 减少5点防御力（不常用）
ATK+15,DEF+10       # 多个属性用逗号分隔
HP+50,MP+30         # 可以是任何属性
```

**注册示例**：
```
注册装备 铁制短剑*普通短剑*hand*ATK+15
注册装备 钢铁胸甲*防御胸甲*chest*DEF+20,HP+50
注册装备 王冠*尊贵王冠*head*ATK+5,DEF+5
```

### 装备操作（玩家）

**查看当前装备**：
```
装备
```

显示当前穿着的所有装备及其属性加成。

**示例输出**：
```
⚔️ 张三 的装备:

🎩 头部: 铁制头盔 (DEF+10)
🛡️ 胸部: 皮革甲胄 (DEF+20,HP+50)
⚔️ 手部: 铁制短剑 (ATK+15)
👖 腿部: 空
👢 脚部: 敏捷靴子 (AGI+5)

📊 总属性加成:
· ATK+15
· DEF+30
· AGI+5
· HP+50
```

**穿上装备**：
```
装备 EQUIP_001          # 按代码装备
装备 铁制短剑           # 按名称装备
```

**卸下装备**：
```
脱装备 head   # 卸下头部装备
脱装备 hand   # 卸下手部装备
```

**查看装备列表**：
```
装备列表
```

显示所有可用的装备。

**查看装备详情**：
```
装备详情 EQUIP_SWORD_01
```

### 属性计算示例

**单装备加成**：
```
玩家基础属性：ATK 40
装备：铁制短剑 (ATK+15)
总属性：ATK 40 + 15 = ATK 55
```

**多装备加成**：
```

装备 1：铁制短剑 (ATK+15)
装备 2：皮革甲胄 (DEF+20, HP+50)
装备 3：敏捷靴子 (AGI+5)

总属性：
- ATK: 40 + 15 = 55
- DEF: 30 + 20 = 50
- HP: 80 + 50 = 130
- AGI: 0 + 5 = 5
```

## 池子系统

### 概念说明

池子是一个虚拟容器，用于管理随机掉落和抽取。支持两种池子类型：

**固定池**（加权随机，不减少数量）：
- 适合：常驻抽取池、月卡池等
- 每次抽取不会减少物品数量

**自由池**（有限数量，抽完即止）：
- 适合：限量活动奖励、单次副本掉落等
- 抽完即止，需管理员重新补充

### 创建池子

**固定池**：
```
注册池子 池子名 fixed
```

**自由池**：
```
注册池子 池子名 free
```

**示例**：
```
注册池子 十连池 fixed
注册池子 活动奖励池 free
```

### 向池子添加物品

**固定池**（需指定权重）：
```
上架池子 池子名 物品码*权重
```
权重越高，被抽中概率越大。

**示例**：
```
上架池子 十连池 ITEM_001*50
上架池子 十连池 ITEM_002*30
上架池子 十连池 ITEM_003*20
```

**自由池**（需指定数量）：
```
上架池子 池子名 物品码*数量
```

**示例**：
```
上架池子 活动奖励池 ITEM_003*10
上架池子 活动奖励池 ITEM_004*5
```

### 从池子移除物品

```
从池移除 池子名 物品码
```

**示例**：
```
从池移除 十连池 ITEM_001
从池移除 活动奖励池 医疗包
```

### 配置每日抽取限制

**查看配置**：
```
池子设定 查看
```

**全局限制**（所有池统计）：
```
池子设定 总量:10
```

**单个池限制**：
```
池子设定 十连池:5
```

**无限制**：
```
池子设定 总量:无限
```

### 玩家抽取

```
抽取 池子名 [次数]
```
默认抽取1次。

**示例**：
```
抽取 十连池       # 抽取1次
抽取 十连池 10    # 抽取10次
```

### 发放额外抽取次数（管理员）

```
发放抽取 角色名 10
发放抽取 角色名 十连池 5
```
增加玩家的永久额外抽取次数。

### 同步地点池子

```
同步踩点池
```
将地点系统中的所有地点自动创建对应的抽取池。

## 合成系统

### 注册合成配方

```
上传配方 产物名*材料名:数量,材料名:数量[*成功率]
```

**参数**：
- **产物名**：合成结果物品的名称
- **材料**：格式为 `物品名:数量`，多个用逗号分隔
- **成功率**（可选）：
  - 范围：0-100
  - 默认值：100（100%成功）
  - 失败时消耗材料但不获得产物

**示例**：
```
。上传配方 简易绷带*干净的布:2,酒精:1
（默认100%成功）

。上传配方 高级丹*初级丹:3,金币:100*80
（80%成功率）

。上传配方 传奇装备*高级装备:5,魔晶石:10*60
（60%成功率）
```

### 查看配方

```
。查看配方
```
玩家命令，查看所有已注册的配方。

### 执行合成

```
。合成 物品名 [数量]
```

**合成结果**：
- ✅ **全部成功**：制作了 [物品] x3
- ❌ **全部失败**：消耗了材料，但未能制作出 [物品]
- ⚠️ **部分成功**：制作了 [物品] x2/3（批量合成时）

**说明**：
- 每次合成单独计算成功率
- 失败时材料仍会消耗
- 支持批量合成 `。合成 物品名 3`

## 攻防系统

### 系统概览

攻防（PK）系统是一个完整的回合制多人战斗系统，支持2+名玩家的自由混战。

### 战斗属性

每个玩家拥有以下核心属性：

| 属性 | 英文 | 说明 | 默认值 |
|------|------|------|--------|
| 生命值 | HP | 玩家的健康值 | 100 |
| 攻击力 | ATK | 每次攻击造成的基础伤害 | 50 |
| 防御力 | DEF | 减少受到的伤害 | 30 |
| 敏捷 | AGI | 决定行动顺序 | 40 |
| 魔法值 | MP | 使用技能的资源 | 50 |
| 恢复速率 | MP_REGEN | 每回合恢复的魔法值 | 5 |
| 临时护盾 | TMP_SHIELD | 优先受伤的盾值 | 0 |

**属性来源**：
- 基础属性：角色初始值
- 装备加成：穿着装备后的增加值
- 物品效果：使用属性物品的加成

### 玩家命令

#### 战斗管理

```
PK 发起                      # 发起开放战斗（其他人可加入）
PK 发起 玩家名 玩家名...     # 指定对手发起战斗（混战模式）
PK 接受 <战斗ID>             # 加入一场战斗
PK 拒绝 <战斗ID>             # 拒绝参加战斗
```

#### 战斗操作（轮到你时可用）

```
攻击 <对手名字>              # 发动普通攻击
防守                         # 进入防守姿态（增加防御）
投降                         # 尝试逃跑（有成功率限制）
逃跑                         # 同投降
```

#### 信息查询

```
属性                         # 查看自己的战斗属性
属性 <玩家名>               # 查看其他玩家的属性
战斗状态                     # 查看当前战斗情况
战斗状态 <战斗ID>           # 查看指定战斗情况
战斗历史 <战斗ID> [页码]    # 查看战斗行动记录
设置昵称 <昵称>             # 设置战斗中显示的昵称
```

### 管理员命令

#### 系统控制

```
攻防 开                      # 启用攻防系统
攻防 关                      # 禁用攻防系统
攻防 查看                    # 查看当前所有配置
攻防 设置 参数 值            # 修改参数配置
```

#### 可配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 每日发起 | 每位玩家每天最多发起的战斗次数 | 10 |
| 每日拒绝 | 每位玩家每天最多拒绝的战斗次数 | 10 |
| 回合超时 | 单个回合的超时时间（毫秒） | 3600000 |
| 默认回合 | 新战斗的默认回合数 | 10 |
| 逃脱率 | 投降逃脱的成功概率(0-100%) | 30 |
| 伤害随机 | 伤害值随机性（如"0-20"） | 0 |
| 强制参战 | 是否强制参加战斗 | 否 |
| 最小人数 | 开始战斗的最少玩家数 | 2 |
| 手动开始 | 是否需要管理员手动启动战斗 | 否 |

#### 玩家管理

```
添加人员 <玩家名> [玩家2]... # 手动创建战斗并加入指定玩家
添加技能 <玩家名> <技能名>   # 为玩家解锁技能
属性 设置 <玩家> <属性> <值>  # 修改玩家的战斗属性
```

### 战斗机制

#### 初始化流程

1. 玩家发起战斗 → 战斗进入"待接受"状态
2. 其他玩家接受参加 → 战斗进入"准备中"状态
3. 达到最少人数或管理员启动 → 战斗进入"进行中"状态

#### 回合顺序

- 按**敏捷(AGI)从高到低**排列行动顺序
- 每个回合轮流操作
- 击败的玩家自动跳过

#### 伤害计算

```
基础伤害 = 攻击者ATK
防御减伤 = 防守者DEF * 0.1
最终伤害 = max(1, 基础伤害 - 防御减伤 + 随机值)
```

**伤害应用顺序**：
1. 优先扣除护盾 (TMP_SHIELD)
2. 护盾不足则扣除生命值 (HP)

#### 逃脱机制

- 成功率可配置（默认30%）
- 失败则继续战斗

## 常见问题

### 属性和装备

**Q: 装备后会自动增加属性吗？**
A: 是的，穿上装备后属性会立即增加，脱下后立即减少。

**Q: 能同时穿多件相同装备吗？**
A: 不能。每个槽位只能穿一件装备。

**Q: 装备会损坏或掉耐久度吗？**
A: 不会。装备可以无限使用。

### 物品和使用次数

**Q: 使用次数为0和-1有什么区别？**
A: 
- `0`：一次性物品，使用后立即消失
- `-1`：无限次数，永不消失

**Q: 物品和互动物品有什么区别？**
A: 
- 物品：玩家对自己使用（`普通使用`）
- 互动物品：玩家对他人使用（`施加`）

### 池子和抽取

**Q: 固定池和自由池的区别？**
A:
- 固定池：加权随机，不减少物品，适合常驻池
- 自由池：有限数量，抽完即止，适合活动奖励

**Q: 如何重置玩家的抽取次数？**
A: 新游戏日会自动重置，或管理员可重新设定限制。

### 货币和经济

**Q: 如何避免经济崩溃？**
A:
1. 监测玩家货币量
2. 调整商城价格
3. 调整物品产出源（池子、合成）
4. 增加消耗渠道（强化、合成）

**Q: 金币和钻石应该如何定价？**
A:
- 金币：低价值，易获得，消耗快
- 钻石：高价值，难获得，消耗少

### 管理和维护

**Q: 应该先做什么？**
A: 推荐顺序：属性 → 货币 → 物品 → 装备 → 池子 → 商城 → 合成 → 攻防

**Q: 能否修改已注册的物品/装备？**
A: 目前系统不支持直接修改。建议注册新物品/装备而不是修改旧的。

**Q: 能否删除已注册的物品？**
A: 目前不支持删除，建议停用而非删除以保持向后兼容。

**Q: 二手市场手续费应该设多少？**
A: 推荐 2-3% 防止炒作，不过高。

### 战斗和属性

**Q: 装备属性加成在战斗中生效吗？**
A: 是的。战斗中使用的属性是装备后的值。

**Q: 防御力如何减少伤害？**
A: 每1点防御力减少10%的伤害（DEF×0.1）。

**Q: 能设置强制参加战斗吗？**
A: 可以，执行 `攻防 设置 强制参战 是`。

## 最佳实践

### 物品设计

1. **合理使用次数**
   - 消耗品：0（一次性）
   - 道具：-1（无限）
   - 任务物品：-1（无限，防止丢失）

2. **二手市场策略**
   - 允许：普通物品、装备、消耗品
   - 禁止：任务物品、限定物品、系统物品

### 装备设计

1. **槽位利用**
   - 充分利用5个槽位
   - 让不同部位都有装备

2. **属性平衡**
   - 同槽位装备属性差异不过大
   - 多属性装备比单属性更有价值

### 经济平衡

1. **监控指标**
   - 玩家平均货币量
   - 物品流通速度
   - 池子抽取频率

2. **调整机制**
   - 调整商城价格
   - 修改池子概率
   - 增删消耗渠道

### 日常维护

- **每周检查**：物品列表、池子配置、市场价格
- **定期备份**：存储数据、配置信息
- **关注反馈**：玩家建议、平衡问题
