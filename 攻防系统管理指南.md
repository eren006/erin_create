# ⚔️ 攻防系统管理指南

本指南面向管理员，介绍如何配置和管理攻防系统。

## 一、系统启用/禁用

### 启用系统
```
攻防 开
```
启用后，玩家才能进行PK战斗。

### 禁用系统
```
攻防 关
```
禁用后，所有PK相关命令将被阻止。

## 二、配置管理

### 查看当前配置
```
攻防 查看
```
显示所有当前的配置参数。

### 修改配置参数
```
攻防 设置 <参数名> <值>
```

## 三、可配置参数详解

### 1. 每日发起次数限制
```
攻防 设置 每日发起 10
```
- **说明**: 每位玩家每天最多可以发起战斗的次数
- **默认值**: 10
- **建议值**:
  - 竞技模式: 5（限制战斗数量，保持紧张感）
  - 休闲模式: 20（允许更多参与）
  - 竞速模式: 10（平衡）
- **应用场景**: 防止玩家过度发起战斗，控制游戏节奏

### 2. 每日拒绝次数限制
```
攻防 设置 每日拒绝 10
```
- **说明**: 每位玩家每天最多可以拒绝战斗的次数
- **默认值**: 10
- **建议值**:
  - 竞技模式: 3（鼓励接受战斗）
  - 休闲模式: 20（玩家自由选择）
  - 竞速模式: 10（平衡）
- **应用场景**: 阻止玩家无限拒绝，鼓励参与

### 3. 单个回合超时时间
```
攻防 设置 回合超时 3600000
```
- **说明**: 玩家在单个回合内的操作超时时间（毫秒）
- **默认值**: 3600000（1小时）
- **常用值**:
  - 竞技模式: 300000（5分钟）- 快速决出
  - 休闲模式: 3600000（1小时）- 玩家有充足时间
  - 竞速模式: 60000（1分钟）- 快速节奏
  - 测试/演示: 30000（30秒）
- **应用场景**: 防止AFK（挂机），确保战斗流畅进行

### 4. 默认回合数
```
攻防 设置 默认回合 10
```
- **说明**: 新创建战斗的初始回合数
- **默认值**: 10
- **建议值**:
  - 竞技模式: 10-15（充分对抗）
  - 休闲模式: 10（适中）
  - 竞速模式: 5（快速决出）
- **应用场景**: 决定战斗长度，影响游戏节奏

### 5. 逃脱成功率
```
攻防 设置 逃脱率 30
```
- **说明**: 玩家投降/逃跑成功的概率（0-100%）
- **默认值**: 30%
- **建议值**:
  - 竞技模式: 10%（逃跑困难，鼓励硬碰硬）
  - 休闲模式: 50%（平衡概率）
  - 竞速模式: 30%（中等难度）
  - 高难度: 5%（极难逃脱）
  - 简单模式: 70%（容易逃脱）
- **应用场景**: 调整战斗难度和逃脱难度

### 6. 伤害随机性
```
攻防 设置 伤害随机 0
```
- **说明**: 攻击伤害的随机性配置
- **配置选项**:
  - `0` - 纯数值（无随机，伤害固定）
  - `10-20` - 范围随机（在纯伤害基础上+10到+20）
  - `5-15` - 更小范围
  - `0-30` - 更大范围
- **建议方案**:
  - 竞技模式: `0`（纯数值，计算准确，策略性强）
  - 休闲模式: `10-20`（增加变数，刺激性强）
  - 竞速模式: `5-15`（平衡随机性和策略）
- **应用场景**: 
  - 纯数值：竞技/PvP环境，重视策略
  - 带随机：休闲环境，增加运气成分

### 7. 强制参战模式
```
攻防 设置 强制参战 否
```
- **说明**: 是否启用强制参战（无法拒绝战斗）
- **可选值**: `是` / `否`
- **默认值**: `否`（允许拒绝）
- **应用场景**:
  - 否（推荐）：玩家自由选择参加，更尊重玩家
  - 是：完全竞技模式，所有挑战必须接受
- **注意**: 启用后，拒绝次数限制失效

### 8. 最小参战人数
```
攻防 设置 最小人数 2
```
- **说明**: 战斗自动开始所需的最少玩家数
- **默认值**: 2（1v1）
- **可选值**: 2, 3, 4, ...
- **应用场景**:
  - 2: 允许1v1对抗（推荐）
  - 3+: 强制多人混战
- **配合**: 可配合"手动开始"参数

### 9. 手动开始模式
```
攻防 设置 手动开始 否
```
- **说明**: 是否需要管理员手动启动战斗（不自动开始）
- **可选值**: `是` / `否`
- **默认值**: `否`（自动开始）
- **应用场景**:
  - 否（推荐）：玩家发起后自动开始，流畅
  - 是：需要管理员审核/手动启动，有控制力
- **配合**: 与"最小人数"配合使用

## 四、预设配置方案

### 方案1: 竞技模式 🏅
```
攻防 设置 每日发起 5
攻防 设置 每日拒绝 3

攻防 设置 默认回合 10
攻防 设置 逃脱率 10
攻防 设置 伤害随机 0
攻防 设置 强制参战 否
攻防 设置 最小人数 2
攻防 设置 手动开始 否
```

**特点**: 严肃竞技环境，限制多，要求硬碰硬
- 每日发起次数少，保证质量
- 拒绝次数少，鼓励参与
- 回合超时短，避免卡顿
- 伤害纯数值，策略性强
- 逃脱难度低，鼓励坚持

### 方案2: 休闲模式 🎮
```
攻防 设置 每日发起 20
攻防 设置 每日拒绝 20
攻防 设置 回合超时 3600000
攻防 设置 默认回合 10
攻防 设置 逃脱率 50
攻防 设置 伤害随机 10-20
攻防 设置 强制参战 否
攻防 设置 最小人数 2
攻防 设置 手动开始 否
```

**特点**: 轻松娱乐环境，限制少，运气成分多
- 每日次数多，充分游玩
- 拒绝次数多，玩家自由选择
- 回合超时长，充足操作时间
- 伤害有随机，增加刺激
- 逃脱概率高，容易脱身

### 方案3: 竞速模式 ⚡
```
攻防 设置 每日发起 10
攻防 设置 每日拒绝 5
攻防 设置 回合超时 60000
攻防 设置 默认回合 5
攻防 设置 逃脱率 20
攻防 设置 伤害随机 5-15
攻防 设置 强制参战 否
攻防 设置 最小人数 2
攻防 设置 手动开始 否
```

**特点**: 快速对决环境，回合少，时间紧
- 回合数少，快速结束
- 超时时间短，节奏快
- 伤害适度随机，平衡策略和运气
- 拒绝次数适中，鼓励参与

### 方案4: 高难度模式 💀
```
攻防 设置 逃脱率 5
攻防 设置 伤害随机 0
攻防 设置 最小人数 3
```

**特点**: 极难模式
- 逃脱几乎不可能
- 伤害计算精确
- 强制多人混战

## 五、玩家管理

### 手动创建战斗
```
添加人员 张三 李四 王五
```
管理员可以直接创建战斗并指定参战玩家，无需玩家发起。

**用途**:
- 组织比赛
- 强制玩家参加
- 快速测试功能

### 解锁技能
```
添加技能 张三 旋风斩
添加技能 李四 治疗术
```
为特定玩家解锁技能（Phase 2功能）。

**注意**: 技能系统目前处于准备阶段，暂未实装。

### 修改玩家属性
```
属性 设置 张三 ATK 80
属性 设置 李四 DEF 50
属性 设置 王五 HP 150
```

**可修改属性**:
- `ATK` - 攻击力
- `DEF` - 防御力
- `AGI` - 敏捷
- `HP` - 生命值
- `TMP_SHIELD` - 临时护盾
- `MP` - 魔法值
- `MP_REGEN` - 每回合恢复

**用途**:
- 给予特殊玩家加强
- 修复BUG后恢复状态
- 进行平衡调整
- 奖励或惩罚

## 六、管理最佳实践

### 新服务器初始设置
1. 启用系统: `攻防 开`
2. 选择模式（推荐休闲): 复制对应配置方案
3. 测试: `添加人员 <玩家1> <玩家2>` 创建测试战斗

### 日常维护
- **每周检查**: `攻防 查看` 确认配置未异常
- **问题调整**: 根据玩家反馈调整参数
- **数据维护**: 定期检查战斗历史，清理长期不用账户

### 遇到问题时的调试
| 问题 | 可能原因 | 解决方案 |
|------|--------|--------|
| 战斗卡顿 | 超时时间太长 | 减少回合超时时间 |
| 玩家不参加 | 拒绝限制少或太少激励 | 增加逃脱难度或拒绝次数 |
| 战斗过短 | 回合数太少 | 增加默认回合数 |
| 伤害差异大 | 随机性范围过大 | 减小或关闭随机性 |
| 玩家投诉逃脱难 | 逃脱率太低 | 调高逃脱率 |

### 模式转换示例
从竞技→休闲:
```
攻防 设置 每日发起 20
攻防 设置 每日拒绝 20
攻防 设置 回合超时 3600000
攻防 设置 伤害随机 10-20
攻防 设置 逃脱率 50
```

## 七、常见配置问题

**Q: 新服务器应该用哪个模式？**
A: 推荐休闲模式，让玩家自由体验。等反馈充足后再调整。

**Q: 如何找到"平衡"的配置？**
A: 从推荐方案开始，根据：
- 玩家反馈调整逃脱率
- 玩家活跃度调整发起/拒绝次数
- 游戏节奏调整回合超时

**Q: 能否对不同玩家设置不同参数？**
A: 目前配置是全局的。个别调整可通过 `属性 设置` 修改玩家属性。

**Q: 伤害随机性应该怎么选？**
A: 
- 竞技/PvP: 用 `0`（纯数值，公平）
- 休闲: 用 `10-20`（增加乐趣）
- 不确定: 用 `5-15`（平衡）

## 八、配置对游戏的影响矩阵

| 参数 | ↑ 增加 | ↓ 降低 |
|------|--------|--------|
| 每日发起 | 战斗增多 | 战斗减少 |
| 每日拒绝 | 玩家自由度高 | 玩家被迫参加 |
| 回合超时 | 时间充足，AFK减少 | 时间紧张，节奏快 |
| 默认回合 | 战斗长 | 战斗短 |
| 逃脱率 | 容易逃脱 | 难以逃脱 |
| 伤害随机 | 差异大，运气成分多 | 结果稳定，策略性强 |
| 最小人数 | 强制多人混战 | 允许1v1 |

## 九、存储位置
- **存储位置**: 主插件 changriV1 → `attack_defense_config`
- **可查看**: 通过 `攻防 查看` 命令实时查看
- **自动保存**: 每次修改配置自动保存

## 十、权限要求
所有管理命令都需要 **管理员权限**，包括：
- ctx.privilegeLevel = 100（超管）
- 或在主插件的 a_adminList 中

如需了解玩家命令，请参考 `攻防系统命令速查.md`。
