## 核心概念

### 升级系统是什么？

一个**玩家角色等级系统**，所有玩家遵循相同的升级规则。管理员只需配置一套规则，所有玩家自动适用。

### 三个关键概念

| 概念 | 说明 |
|------|------|
| **等级** | 玩家的当前等级（1-100） |
| **消耗品** | 升级时需要消耗的资源（金币、物品、属性等） |
| **奖励品** | 升级成功后获得的奖励（属性、货币、物品等） |

### 升级流程

```
玩家执行升级
  ↓
检查是否满足消耗条件
  ↓
判断成功率（可能失败）
  ↓
消耗资源（即使失败也消耗）
  ↓
发放奖励（仅成功时）
  ↓
更新等级和历史记录
```

## 命令参考

### 管理员命令

#### 上传升级等级 - 配置升级规则

**命令格式**：
```
上传升级等级 <等级|范围> <描述> * <消耗品> * <奖励品> [*成功率]
```

**参数说明**：

| 参数 | 说明 | 示例 |
|------|------|------|
| 等级\|范围 | 单个等级或范围 | `1` 或 `1-50` |
| 描述 | 等级名称（支持{等级}模板） | `新手冒险者` 或 `{等级}级战士` |
| 消耗品 | 升级消耗（逗号分隔） | `金币:100` 或 `金币:50+50,升级石:0+1` |
| 奖励品 | 升级奖励（逗号分隔） | `HP+10` 或 `HP+5+5,金币:50` |
| 成功率 | 升级成功概率（可选，默认100） | `100` 或 `80` |

**示例**：

```bash
# 单个等级配置
上传升级等级 1 * 新手冒险者 * 金币:100 * HP+10,ATK+5

# 批量配置（推荐）
上传升级等级 1-10 * {等级}级冒险者 * 金币:50+50 * HP+5+5,ATK+3+2

# 带成功率
上传升级等级 20-30 * {等级}级战士 * 金币:500+100,晶体:1+1 * ATK+15+10 * 90
```

## 使用示例

### 消耗品配置

支持4种消耗品类型，任意组合：

| 类型 | 格式 | 示例 |
|------|------|------|
| 物品 | `物品名:数量` | `升级石:2` 或 `升级石:0+1` |
| 互动物品 | `互动物品名:数量` | `魔法卷轴:1` 或 `魔法卷轴:1+1` |
| 货币 | `货币名:数量` | `金币:100` 或 `金币:50+50` |
| 属性 | `属性名:数值` | `HP:50` 或 `HP:20+10` |

**示例**：
```
金币:100                          # 消耗100金币
金币:50+50                        # 消耗：1级50, 2级100, 3级150...
升级石:2,金币:100                # 消耗2个升级石和100金币
金币:500,升级石:0+1              # 消耗500金币，升级石逐级递增
HP:50消耗                         # 消耗50点HP（献祭式升级）
```

---

### 奖励品配置

支持3种奖励品类型，任意组合：

| 类型 | 格式 | 示例 |
|------|------|------|
| 属性 | `属性名+数值` | `HP+10` 或 `HP+5+5` |
| 货币 | `货币名:数量` | `金币:50` 或 `金币:50+50` |
| 物品 | `物品名:数量` | `升级证:1` 或 `升级证:1+0` |

**示例**：
```
HP+10                             # 增加10点HP
HP+5+5                            # 1级+5, 2级+10, 3级+15...
金币:100                          # 获得100金币
HP+10,金币:50,升级证:1           # 复合奖励
（无奖励）                        # 升级后无任何奖励
```

---

### 递增公式

```
第N级值 = 基础 + (N-1) × 增幅
```

**示例计算**：
```
配置: 金币:50+50
  1级: 50 + (1-1) × 50 = 50
  2级: 50 + (2-1) × 50 = 100
  3级: 50 + (3-1) × 50 = 150
  ...
  10级: 50 + (10-1) × 50 = 500

配置: HP+5+5
  1级: +5
  2级: +10
  3级: +15
  ...
  10级: +50
```

## 常见问题

### Q1: 可以跳级升级吗？
**A**: 不可以。必须逐级升级。但支持连续升级（可以一次升多级，实现方式是在客户端多次执行升级命令）。

### Q2: 升级失败会消耗材料吗？
**A**: 会的。失败时材料仍会被消耗。成功率100%时才不会失败。

### Q3: 如何修改已配置的等级？
**A**: 重新执行相同的等级范围命令即可覆盖旧配置。例如重新执行 `上传升级等级 1-10 ...` 会覆盖1-10级的所有配置。

### Q4: 消耗品和奖励品可以为空吗？
**A**: 可以的。消耗品为空表示升级无需消耗资源，奖励品为空表示升级后无任何奖励（纯装饰等级）。

### Q5: {等级} 占位符会自动替换吗？
**A**: 是的。`{等级}` 会被替换为实际等级号。比如第2级时，`{等级}级冒险者` 变成 `2级冒险者`。

### Q6: 成功率是什么？
**A**: 升级的成功概率（0-100%）。失败时仍消耗材料但不升级，不发放奖励。

### Q7: 多个职业可以有不同的升级规则吗？
**A**: 目前升级规则是全局统一的。如果需要职业差异，建议通过消耗品和奖励品的设计来实现（如不同职业消耗不同的材料）。

### Q8: 最大等级可以改吗？
**A**: 可以。在规则配置中有 `max_level` 字段，默认100。

## 最佳实践

```bash

```

前期只用金币，后期引入道具：

```bash
# 前期
上传升级等级 1-20 * {等级}级 * 金币:50+50 * HP+5+5

# 中期引入道具
上传升级等级 21-40 * {等级}级 * 金币:300+100,升级石:0+1 * HP+15+10,ATK+10+5
```
