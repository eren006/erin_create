## 系统概述

### 功能介绍
长日设置系统是长日恋综生态的统一控制台，与主插件 `changriV1` 和 `长日物品.js` 配合使用，提供：

- 📺 **统一设置面板**：10大模块配置
- ⏰ **天数管理**：游戏时间线推进与自动推进
- 📊 **统计报告**：实时互动数据统计
- 🔗 **数据关联**：与物品、装备、关系线等系统联动

### 系统架构
```
长日系统 (changriV1) - 主系统，管理人物、关系、互动
  ↓
长日物品.js - 物品、装备、货币系统
  ↓
长日设置.js - 配置控制台 ← 你在这里
```

### 权限要求
- 所有设置操作均需**管理员权限**
- 管理员由主插件统一管理（存储键：`a_adminList`）

## 快速开始

### 基本操作流程

**第一步：查看设置**
```
.设置 基础
```
机器人会返回当前的所有设置，格式如：
```
.设置 基础设置
【恋综名】xxx恋综
【微信】开启
【礼物】开启
...
```

**第二步：复制并编辑模板**
- 复制整个返回的消息（包括第一行）
- 只修改你想改变的 【参数】 的值
- 其他参数可保持不变

**第三步：粘贴提交**
- 粘贴编辑后的完整模板
- 机器人自动识别第一行标识符（如"设置 基础设置"）
- 处理修改请求

## 互动设置

### 作用范围
配置玩家在约会、赠礼、寄信等互动中的冷却时间和参数。

### 可配置参数

| 参数 | 说明 | 默认值 | 范围 | 建议 |
|------|------|--------|------|------|
| **地点系统** | 约会是否需要选择地点 | 开启 | 开启/关闭 | 开启更有沉浸感 |
| **结戏抽取** | 约会结束是否触发特殊抽取 | 开启 | 开启/关闭 | 看游戏风格 |
| **电话最小时长** | 电话约会最短时间(分钟) | 29 | 1-120 | 25-30较合理 |
| **私密最小时长** | 私密约会最短时间(分钟) | 59 | 30-240 | 50-60较合理 |
| **寄信冷却时间** | 两次寄信间隔(秒) | 60 | 10-600 | 30-60防刷屏 |
| **送礼冷却时间** | 两次赠礼间隔(秒) | 30 | 10-300 | 20-30合理 |
| **送礼模式** | 冷却计算方式 | 0 | 0 / 1 | 0为标准 |

### 参数说明

**地点系统**
- 开启：约会时玩家需选择地点，增加代入感
- 关闭：无地点选择，约会更简化

**电话 vs 私密时长**
- 电话：可以是比较短的互动
- 私密：通常应该更长，代表更亲密的互动
- 建议：私密时长 ≥ 电话时长

**冷却时间**
- 防止玩家频繁刷屏和娱乐内容重复
- 时间太长会影响游戏活跃度
- 时间太短容易被滥用

**送礼模式**
- `0`：只计算赠送者冷却
- `1`：计算双向冷却（双方都有冷却）

### 修改示例
```
.设置 互动设置
【地点系统】开启
【结戏抽取】开启
【电话最小时长】25
【私密最小时长】55
【寄信冷却时间】45
【送礼冷却时间】25
【送礼模式】0
```

## 物品系统设置

### 作用范围
与 `长日物品.js` 联动，配置物品系统的相关参数。

### 查看设置
```
.设置 道具
```

### 可配置参数

| 参数 | 说明 | 范围 | 用途 |
|------|------|------|------|
| **追踪器成功率** | SPEC_001追踪器的成功概率 | 0-100 | 追踪能否成功 |
| **每日抽取上限** | 每天最多能抽取物品次数 | ≥1的整数 | 防止刷物品 |
| **追踪器显示伙伴** | 追踪结果是否显示伙伴信息 | 开启/关闭 | 信息透明度 |
| **追踪器时间限制** | 追踪器是否受时间限制 | 开启/关闭 | 约束条件 |
| **物品池模式** | 物品抽取池的模式 | 自由池/固定池 | 抽取规则 |
| **施加是否提醒** | 使用互动物品时是否提醒 | 开启/关闭 | 体验参数 |
| **暴露名字概率** | 施加互动物品时泄露使用者身份的概率 | 0-100 | 神秘性 |
| **施加可用时段** | 互动物品的可用时段 | 不限/时间段 | 时间限制 |

### 参数说明

**物品池模式**：
- `自由池`：允许任意物品组合
- `固定池`：预设物品池

**施加可用时段**：
- `不限` 或 `全部`：全天可用
- 格式：`18-23` 表示晚上6点到11点
- 多段：`9-12,14-18` 表示9-12点和14-18点

### 修改示例
```
.设置 道具设置
【追踪器成功率】75
【每日抽取上限】2
【追踪器显示伙伴】开启
【追踪器时间限制】开启
【物品池模式】自由池
【施加是否提醒】开启
【暴露名字概率】20
【施加可用时段】18-23
```

## 群组与目击设置

### 作用范围
管理小群的生命周期、私约规则和目击报告系统。

### 查看设置
```
.设置 群组
```

### 可配置参数

#### 小群管理

| 参数 | 说明 | 范围 | 用途 |
|------|------|------|------|
| **小群过期时间** | 小群自动过期的时间 | 数字(小时) | 群号回收周期 |
| **允许加入已有私约** | 是否允许加入进行中的约会 | 开启/关闭 | 私约灵活性 |
| **自动合并重合私约** | 时间重叠的私约是否自动合并 | 开启/关闭 | 避免冲突 |
| **复盘强制结束** | 是否必须复盘才能结束约会 | 开启/关闭 | 约会流程要求 |

#### 目击报告系统

| 参数 | 说明 | 范围 | 用途 |
|------|------|------|------|
| **目击报告** | 是否启用目击报告功能 | 开启/关闭 | 全局开关 |
| **目击每日上限** | 每天最多报告次数 | 数字 | 防止刷屏 |
| **时间重叠阈值** | 判定"同时出现"的时间重叠率 | 0-100% | 灵敏度 |
| **目击报告方式** | 报告如何发送 | 双向通知/单向通知 | 信息对称性 |
| **包含已结束** | 已结束的约会是否纳入目击 | 是/否 | 范围设定 |

#### 参数说明

**目击报告系统**：
- 用于跟踪玩家在游戏中的"目击"互动
- 时间重叠：如两人都在14:00-15:00约会，则有100%重叠
- 阈值30%：表示重叠部分≥30%则触发目击

**目击报告方式**：
- `双向通知`：A看到B，B也看到A
- `单向通知`：只有主动报告者看到信息

### 修改示例

```
.设置 群组设置
【小群过期时间】48
【允许加入已有私约】开启
【自动合并重合私约】关闭
【复盘强制结束】开启
【目击报告】开启
【目击每日上限】5
【时间重叠阈值】30
```

## 天数系统

### 功能说明
管理游戏的时间线推进，所有日常限制（如寄信上限）都按天数重置。

### 天数格式
所有天数以 `D数字` 格式表示：
- `D0` - 第0天
- `D1` - 第1天
- `D2` - 第2天
- ...以此类推

### 手动推进天数

#### 命令
```
.设置天数 D<数字>
```

#### 执行流程
1. **生成报告** ← 基于清空前的实时数据
2. 清空所有计数 → 清零为 0 或 []
3. 设置新天数
4. 发送报告到后台群

#### 实际效果
- ✅ 天数更新为设置的新天数
- ✅ 所有互动计数被清零（电话、私密、寄信等）
- ✅ 心愿池清空
- ✅ 心动信池清空
- ✅ 每日寄信限制重置
- ✅ 统计报告准确显示前一天的数据

#### 示例
```
.设置天数 D1    # 推进到第1天
.设置天数 D3    # 推进到第3天
.设置天数 D10   # 推进到第10天
```

### 自动天数推进

#### 启用自动推进
```
.开启自动天数
```
- 每天 23:59 自动推进天数 +1
- 所有计数同样自动清零

#### 关闭自动推进
```
.关闭自动天数
```

### 天数规划建议

**单周制（示例）**：
- D0：开局设置期
- D1-D5：主线剧情期
- D6：剧情高潮
- D7-D10：结局收尾

**多周制（示例）**：
- 每7天为一个周期
- D0-D6：第一周
- D7-D13：第二周
- ...依次递推

## 统计报告

### 自动生成与发送

- 自动推进天数 (`.开启自动天数` 触发)

### 报告内容

统计报告包含以下数据：

#### 1. 基本信息
- 天数变化：`前天数 → 新天数`
- 生成时间
- 是否清空计数

#### 2. 玩家统计
- 绑定角色总数

#### 3. 互动统计
- 电话约会次数
- 私密约会次数
- 官方约会次数
- 寄信次数
- 发送信件次数
- 心动信派送次数
- 礼物赠送次数
- 心愿达成次数

- 待处理邀约请求数
- 多人邀约数
- 心愿漂流瓶数

## 常见问题

### Q1：修改设置后没有生效？
**A：** 
1. 确保复制的是**完整模板**（包括第一行标识符）
2. 检查修改的格式：`【参数名】值`（中文方括号）
3. 检查是否有管理员权限
4. 某些设置可能需要玩家重新连接机器人生效

### Q2：怎样判断修改成功了？
**A：** 机器人会返回：
```
✅ 处理完成（成功 X 项）
【参数名】已更新为 xxx
```

### Q3：一次可以修改多个参数吗？
**A：** 可以。复制模板后，修改多个 【参数】 的值，然后一起粘贴提交。

### Q4：群号怎样填写？
**A：** 
- 直接填写纯数字的群号
- 例如：`【公告群】123456789`
- 不需要平台前缀

### Q5：天数推进后数据会不会丢失？
**A：** 
- ✅ 玩家的绑定数据（角色、关系线等）不会丢失
- ❌ 当日的互动计数会清零（这是设计如此）
- ❌ 心愿池和心动信池会清空（需要重新投稿）

### Q6：混乱信件的概率是怎样计算的？
**A：** 
- 每个异常都独立判定，满足概率就触发
- 一封信可能同时触发多个异常
- 例：送错 10% + 涂改 10% = 10% 送错 或/且 10% 涂改

### Q7：电话和私密时长太长玩家会不会不愿意玩？
**A：** 
- 过长确实会降低活跃度
- 建议根据玩家反馈调整
- 电话 25-30 分钟，私密 50-70 分钟较为平衡

### Q8：如何配置不同天数的不同上限？
**A：** 
仅 **心动信** 支持按天数配置：
```
.设置信箱上限 D0:1 D1:5 D2:3
```

其他功能的上限全局统一。

### Q9：为什么后台群收不到报告？
**A：**
1. 检查 【后台群】 是否设置了正确的群号
2. 检查机器人是否在该群中
3. 检查是否手动推进或启用了自动推进

### Q10：如何查看群组设置？
**A：**
```
.设置 群组
```
可查看和配置小群管理、目击报告等功能。

## 最佳实践

### 1. 初始化流程
```
1. 加载主插件 changriV1
2. 加载物品系统 长日物品.js
3. 加载设置系统 长日设置.js
4. 依次配置：基础 → 互动 → 信件相关 → 公告 → 天数
5. 验证各模块设置已正确保存
6. 启动游戏
```

### 2. 日常管理
- 每天推进天数（手动或自动）
- 定期查看统计报告，了解游戏状态
- 监控混乱信件的发生率
- 根据玩家反馈调整冷却时间

### 3. 周期性维护
- 每周检查群组状态
- 清理过期群组
- 处理待办请求
- 备份重要配置

### 4. 性能考虑
- 不要设置过多的群组
- 定期清理过期数据
- 避免频繁修改配置
- 自动推进建议启用23:59时的推进
