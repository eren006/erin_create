# 升级系统 - 快速开始指南

## 📋 已集成到长日物品.js

升级系统现已添加到长日物品系统中，包含以下命令：

## 管理员命令

### 1. 配置升级规则

**命令格式**：
```
上传升级等级 <等级|范围> <描述> * <消耗品> * <奖励品> [*成功率]
```

**单个等级配置**：
```
上传升级等级 1 * 新手冒险者 * 金币:100 * HP+10,ATK+5
```

**批量配置（推荐）**：
```
上传升级等级 1-10 * {等级}级冒险者 * 金币:50+50 * HP+5+5,ATK+3+2
```

**带成功率**：
```
上传升级等级 20-30 * {等级}级战士 * 金币:500+100,升级石:1+1 * ATK+10+5 * 90
```

### 2. 查看配置

```
升级列表              # 查看所有已配置的等级
查看升级配置 1        # 查看第1级的详细配置
查看升级配置 20       # 查看第20级的详细配置
```

## 快速配置示例

### 方案1：简单升级链（50级）

```
上传升级等级 1-50 * {等级}级冒险者 * 金币:100+100 * HP+10+10,ATK+5+5
```

**效果**：
- 1级：消耗金币100，获得HP+10,ATK+5
- 2级：消耗金币200，获得HP+20,ATK+10
- ...
- 50级：消耗金币5100，获得HP+510,ATK+255

### 方案2：分阶段升级

```bash
# 前期：简单（1-20级）
上传升级等级 1-20 * 新手{等级} * 金币:50+50 * HP+5+5

# 中期：需要道具（21-40级）
上传升级等级 21-40 * 战士{等级} * 金币:300+100,升级石:0+1 * HP+15+10,ATK+10+5

# 后期：高难度（41-60级）
上传升级等级 41-60 * 传奇{等级} * 金币:1000+200,晶体:1+1 * ATK+15+10,DEF+5+3 * 100-70
```

### 方案3：献祭式升级

```
上传升级等级 1-30 * 血之战士{等级} * 金币:100+50,HP:20+10消耗 * ATK+15+5,HP+100+50
```

**说明**：
- 消耗玩家的HP来升级
- 升级时直接从玩家HP中扣除

## 重要说明

### 递增公式

```
第N级值 = 基础 + (N-1) × 增幅

示例：金币:50+50
  1级: 50 + (1-1) × 50 = 50
  2级: 50 + (2-1) × 50 = 100
  3级: 50 + (3-1) × 50 = 150
```

### 消耗品格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `物品名:数量` | 消耗物品或货币 | `金币:100` 或 `升级石:2` |
| `物品名:基础+增幅` | 逐级递增消耗 | `金币:50+50` 或 `升级石:0+1` |
| `属性名:数值` | 消耗属性 | `HP:50` |

### 奖励品格式

| 格式 | 说明 | 示例 |
|------|------|------|
| `属性名+数值` | 增加属性 | `HP+10` 或 `ATK+5` |
| `属性名+基础+增幅` | 逐级递增奖励 | `HP+5+5` 或 `ATK+3+2` |
| `物品名:数量` | 获得物品或货币 | `金币:100` 或 `升级证:1` |

## 常见问题

### Q1: 可以跳级升级吗？
**A**: 不可以。必须逐级升级。但支持批量升级（可以一次升多级）。

### Q2: 升级失败会消耗材料吗？
**A**: 会的。成功率是100%时才不会失败。失败时材料已消耗。

### Q3: 如何修改已配置的等级？
**A**: 重新执行相同的命令即可覆盖旧配置。

### Q4: 消耗品和奖励品可以为空吗？
**A**: 可以的。消耗品为空表示无需消耗资源，奖励品为空表示升级后无奖励。

### Q5: {等级} 占位符会自动替换吗？
**A**: 是的。`{等级}` 会被替换为实际等级号。比如第2级时，`{等级}级冒险者` 变成 `2级冒险者`。

## 系统信息

- **存储键**：
  - `level_up_rules` - 升级规则配置
  - `player_level` - 玩家当前等级
  - `player_level_history` - 玩家升级历史

- **支持递增计算**：消耗品和奖励品都支持自动递增

- **支持成功率**：可配置升级的成功概率（0-100%）

- **与现有系统深度整合**：
  - 直接使用物品、货币、属性系统
  - 升级消耗直接从玩家背包扣除
  - 升级奖励直接加入玩家属性和背包
