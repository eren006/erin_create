# ⚔️ 装备系统管理指南

本指南面向管理员，介绍如何注册、配置和管理装备系统。

## 一、装备注册

### 注册装备命令
```
注册装备 <装备名>*<描述>*<槽位>*<基础属性>*[强化属性]*[最大强化]
```

### 参数说明

| 参数 | 说明 | 示例 |
|------|------|------|
| 装备名 | 显示给玩家的名称 | 铁制短剑 |
| 描述 | 装备说明 | 一把普通的短剑 |
| 槽位 | head/chest/hand/leg/foot | hand |
| 基础属性 | 装备自带的属性加成 | ATK+15,DEF+10 |
| 强化属性 | 每级强化增加的属性 | ATK+2 (可选) |
| 最大强化 | 最多可强化的等级 | 10 (可选，默认10) |

### 属性格式
```
ATK+15          # 增加15点攻击力
DEF-5           # 减少5点防御力（不常用）
ATK+15,DEF+10   # 多个属性用逗号分隔
HP+50,MP+30     # 可以是任何属性
```

### 装备槽位
| 代码 | 名称 | 用途 |
|------|------|------|
| head | 头部 | 头盔、王冠、面罩等 |
| chest | 胸部 | 甲胄、衣服、外套等 |
| hand | 手部 | 武器、盾牌、手套等 |
| leg | 腿部 | 护腿、裤子、靴裤等 |
| foot | 脚部 | 靴子、鞋子等 |

### 注册示例

**示例 1：简单武器**
```
注册装备 铁制短剑*普通短剑*hand*ATK+15*ATK+2*10
```
- 基础加成：ATK+15
- 强化：每级加ATK+2，最多10级
- 满强化后总加成：ATK+35 (15 + 10×2)

**示例 2：防具（多属性）**
```
注册装备 钢铁胸甲*防御胸甲*chest*DEF+20,HP+50*DEF+3*10
```
- 基础加成：DEF+20 和 HP+50
- 强化：DEF+3/级
- 强化不影响HP加成

**示例 3：无强化装备**
```
注册装备 王冠*尊贵王冠*head*ATK+5,DEF+5
```
- 不指定强化属性，表示此装备无法强化
- 仅有基础加成

**示例 4：稀有装备**
```
注册装备 传奇剑*史诗级剑*hand*ATK+50,HP+100*ATK+10*20
```
- 高基础加成
- 可强化到20级（非常规）

## 二、强化系统配置

### 查看强化配置
```
强化 查看
```

### 启用/禁用强化
```
强化 启          # 启用强化系统
强化 禁          # 禁用强化系统
```

### 配置成功率
```
强化 设置 成功率 <0-100>
```

- **默认值**：90%
- **建议值**：
  - 竞技模式：100%（保证成功）
  - 标准模式：90%（略有失败风险）
  - 高难度：70-80%（提高难度）

**示例**：
```
强化 设置 成功率 100    # 强化必定成功
强化 设置 成功率 85     # 15%失败率
```

### 配置消耗
```
强化 设置 消耗 <基础消耗增量>
```

系统根据强化等级自动计算消耗。默认公式：
```
1级：100 + 0×50 = 100金币
2级：100 + 1×50 = 150金币
3级：100 + 2×50 = 200金币
...
10级：100 + 9×50 = 550金币
```

**配置消耗**：
```
强化 设置 消耗 50       # 每级增加50（总消耗1-550）
强化 设置 消耗 100      # 每级增加100（总消耗1-1000）
强化 设置 消耗 20       # 每级增加20（总消耗1-280）
```

## 三、强化配置方案

### 方案 1：竞技模式（推荐）
```
强化 启
强化 设置 成功率 100
强化 设置 消耗 100
```
- 强化必定成功（无失败风险）
- 消耗较高（1-1000金币）
- 快速达到极限，差异来自装备选择

### 方案 2：标准模式
```
强化 启
强化 设置 成功率 90
强化 设置 消耗 50
```
- 有10%失败率，增加不确定性
- 消耗中等（1-550金币）
- 平衡难度和成本

### 方案 3：休闲模式
```
强化 启
强化 设置 成功率 95
强化 设置 消耗 30
```
- 高成功率但有失败可能
- 消耗较低（1-330金币）
- 玩家更容易强化装备

### 方案 4：硬核模式
```
强化 启
强化 设置 成功率 70
强化 设置 消耗 150
```
- 高失败风险
- 消耗巨大（1-1500金币）
- 最终形态装备才能完全强化

## 五、常见管理任务

### 任务 1：新增强力武器
```
注册装备 精钢长剑*强力长剑*hand*ATK+25*ATK+3*10
```
然后上架商城：
```
上架商城 EQUIP_XXX*1000金币
```

### 任务 2：创建防具套装
```
注册装备 龙骑士头盔*龙骑士套装-头*head*DEF+30,ATK+10*DEF+2*15
注册装备 龙骑士铠甲*龙骑士套装-身*chest*DEF+50,HP+100*DEF+5*15
注册装备 龙骑士靴子*龙骑士套装-脚*foot*DEF+20,AGI+10*DEF+2*15
```

### 任务 3：调整难度
如果玩家反馈强化太简单：
```
强化 设置 成功率 80
```

如果反馈强化太难：
```
强化 设置 成功率 95
强化 设置 消耗 30
```

### 任务 4：监测装备平衡
- 检查是否某件装备被所有玩家使用
- 考虑增加竞争装备或削弱该装备
- 不建议修改已注册装备（会失效旧装备），而是注册新装备

## 六、装备与其他系统的集成

### 与物品系统集成
装备可以作为普通物品处理：
1. 注册装备后，可以上架商城
2. 可以添加到池子（抽取系统）
3. 可以作为合成配方的产物
4. 可以通过二手市场交易（如果设置允许）

### 与属性系统集成
装备加成会实时应用到玩家属性：
1. 穿上装备 → 属性立即增加
2. 脱下装备 → 属性立即减少
3. 强化装备 → 属性继续增加

### 与攻防系统集成
装备属性加成应用到战斗：
1. 计算战斗伤害时使用装备后的ATK
2. 计算伤害减少时使用装备后的DEF
3. 计算行动顺序时使用装备后的AGI
4. HP上限和当前HP受装备影响

## 七、数据管理

### 存储位置
- **装备注册表**：`equipment_registry`（主插件存储）
- **玩家装备数据**：`player_equipments`（主插件存储）
- **强化配置**：`equipment_config`（主插件存储）

### 备份建议
定期备份主插件的存储数据，包含所有装备信息。

## 八、常见问题

**Q: 能否修改已注册装备的属性？**
A: 目前系统不支持修改。建议注册新装备而不是修改旧装备。

**Q: 装备是否可以掉落？**
A: 装备作为物品存在，可以通过池子掉落、合成获得、商城购买等方式获取。

**Q: 强化等级有上限吗？**
A: 每件装备有自己的上限（默认10级，可在注册时修改）。

**Q: 能否禁用强化系统？**
A: 可以，执行 `强化 禁`。禁用后玩家无法强化装备。

**Q: 如何处理装备的稀有度或品质？**
A: 目前系统中所有装备平等对待。如果需要品质系统，可以在装备名称中标注（如"[传奇]"）或通过属性差异体现。

**Q: 强化失败时能否返还部分金币？**
A: 目前不支持，失败时全额消耗。这可以通过降低成功率或提高消耗来调整难度。

**Q: 是否支持装备分解或拆解？**
A: 目前不支持，装备只能通过脱掉来卸除。

## 九、扩展建议

后续可以考虑的功能：
- 装备品质系统（普通、稀有、传奇等）
- 套装系统（穿齐套装获得额外加成）
- 装备技能或特效
- 装备掉落表（不同BOSS掉落不同装备）
- 装备交易系统（与二手市场结合）
- 装备重铸（重新分配强化等级）
