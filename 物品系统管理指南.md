# 📦 物品系统管理指南

本指南面向管理员，介绍如何注册、配置和管理物品系统及相关功能。

## 一、物品系统概述

### 系统结构
```
物品系统
├── 物品注册 (ITEM_, INTER_)
├── 货币系统 (CUR_)
├── 属性系统 (Attributes)
├── 商城系统 (Shop)
├── 池子系统 (Pools) - 抽取、合成、踩点
├── 二手市场 (Market)
├── 互动物品 (Interactive Items)
└── 合成系统 (Crafting)
```

### 编码规范（方案1）
```
CUR_xxxx      - 货币 (Currency)
ITEM_xxxx     - 普通物品 (Item)
MARK_xxxx     - 标记/特殊标签 (Mark)
SPEC_xxxx     - 特殊物品 (Special)
INTER_xxxx    - 互动物品 (Interactive)
```

## 二、物品注册

### 2.1 注册普通物品
```
上载物品 物品名*描述*使用次数*属性效果*允许二手
```

**参数说明**:
- **物品名**: 显示给玩家的名称
- **描述**: 物品说明（可选）
- **使用次数**:
  - `-1`: 无限次数（可无限使用）
  - `0`: 一次性（使用后消失）
  - `>0`: 指定次数（使用N次后消失）
- **属性效果** (可选):
  - `属性名+数值` - 增加属性（如 `体力+50`）
  - `属性名-数值` - 减少属性（如 `理智-10`）
  - 多个效果用逗号分隔（如 `体力+50,理智-10`）
- **允许二手**:
  - `Y` 或 `是` - 允许在二手市场转售
  - `N` 或 `否` - 不允许转售（推荐）

**示例**:
```
上载物品 医疗包*恢复生命*3*HP+50*N
上载物品 魔法卷轴*一次性法术*0*MP+30*Y
上载物品 永久属性书*永久增强*-1*ATK+5*N
```

**系统自动编码**:
- 自动生成 `ITEM_001`, `ITEM_002` 等

### 2.2 注册互动物品
```
上载互动物品 物品名*描述*使用次数*属性效果*允许二手
```

**与普通物品的区别**:
- **普通物品**: 玩家对自己使用（使用方式：`普通使用 ITEM_xxx`）
- **互动物品**: 对他人使用（使用方式：`施加 INTER_xxx <目标名>`）

**参数相同**:
```
上载互动物品 治疗术*为他人治疗*5*体力+50*N
上载互动物品 诅咒术*对他人诅咒*1*理智-30*N
```

**系统自动编码**:
- 自动生成 `INTER_001`, `INTER_002` 等

### 2.3 应用物品效果

**普通物品使用**:
```
普通使用 ITEM_001
```
物品效果立即作用于使用者。

**互动物品使用**:
```
施加 INTER_001 李四
```
- 设置玩家匿名选项（通过暴露名字概率控制）
- 可查看其效果配置

**设置互动物品匿名选项**:
```
施加 查看 INTER_001
```
查看该互动物品的配置，包括暴露名字概率。

## 三、货币系统

### 3.1 注册货币
```
注册货币 货币名*描述
```

**参数**:
- **货币名**: 显示名（如"金币"、"点券"）
- **描述**: 货币说明

**示例**:
```
注册货币 金币*通用流通货币
注册货币 钻石*高级货币，需充值获得
注册货币 积分*任务奖励，可兑换物品
```

**系统自动编码**:
- 自动生成 `CUR_001`, `CUR_002` 等

### 3.2 货币操作
```
调整 角色名 CUR_001 +100
调整 角色名 CUR_001 -50
```
直接增加或减少玩家货币数量。

## 四、属性系统

### 4.1 注册属性
```
我创建属性 属性名 [最小值] [最大值] [默认值]
```

**参数**:
- **属性名**: 属性标识（如"体力"、"理智"、"魅力"）
- **最小值**: 属性下限（可选）
- **最大值**: 属性上限（可选）
- **默认值**: 新角色的初始值（可选，默认0）

**示例**:
```
我创建属性 体力 0 100 50
我创建属性 理智 0 100 80
我创建属性 魅力 -50 50 0
我创建属性 金币 0 999999 0
```

### 4.2 查看属性
```
我的状态
```
显示当前角色的所有属性值。

### 4.3 修改属性

**单个修改**:
```
设置属性 角色名 属性名 100
```

**批量修改** (需特殊格式):
```
[使用特殊命令修改]
```

**增减**:
```
体力++ 张三 50
体力-- 张三 30
```
使用 `++` 增加，`--` 减少。

## 五、商城系统

### 5.1 上架物品到商城
```
上架商城 物品码*价格货币名
```

**参数**:
- **物品码**: 物品的编码（如 ITEM_001, CUR_005）
- **价格货币名**: 价格和货币（如 "100金币", "50钻石"）

**示例**:
```
上架商城 ITEM_001*100金币
上架商城 ITEM_005*5钻石
上架商城 CUR_001*10钻石
```

**单位**: 价格必须是整数。

### 5.2 从商城下架
```
商城下架 物品码或物品名
```

**示例**:
```
商城下架 ITEM_001
商城下架 医疗包
```

### 5.3 查看商城
```
商城
```
玩家命令，显示所有可购买物品。

## 六、池子系统（抽取/合成/踩点）

### 6.1 创建池子

**固定池** (加权随机，不减少数量):
```
注册池子 池子名 fixed
```
适合：常驻抽取池、月卡池等

**自由池** (有限数量，抽完即止):
```
注册池子 池子名 free
```
适合：限量活动奖励、单次副本掉落等

**示例**:
```
注册池子 十连池 fixed
注册池子 活动奖励池 free
```

### 6.2 向池子添加物品

**固定池**（需指定权重）:
```
上架池子 池子名 物品码*权重
上架池子 十连池 ITEM_001*50
上架池子 十连池 ITEM_002*30
```
权重越高，被抽中概率越大。

**自由池**（需指定数量）:
```
上架池子 池子名 物品码*数量
上架池子 活动奖励池 ITEM_003*10
上架池子 活动奖励池 ITEM_004*5
```

### 6.3 从池子移除物品
```
从池移除 池子名 物品码或物品名
```

**示例**:
```
从池移除 十连池 ITEM_001
从池移除 活动奖励池 医疗包
```

### 6.4 配置每日抽取次数

**查看配置**:
```
池子设定 查看
```

**全局限制** (所有池统计):
```
池子设定 总量:10
```

**单个池限制**:
```
池子设定 十连池:5
```

**无限制**:
```
池子设定 总量:无限
```

### 6.5 玩家抽取
```
抽取 池子名 [次数]
```
玩家命令，从指定池子抽取。默认1次。

### 6.6 发放额外抽取次数
```
发放抽取 角色名 10
发放抽取 角色名 十连池 5
```
增加玩家的永久额外抽取次数（不随游戏日重置）。

## 七、合成系统

### 7.1 注册合成配方
```
注册合成 产物代码*描述*材料代码1:数量1,材料代码2:数量2[*限制条件]
```

**参数**:
- **产物代码**: 合成结果物品码（如 ITEM_010）
- **描述**: 配方说明（如 "升级丹药"）
- **材料**: 格式为 `物品码:数量`，多个用逗号分隔
- **限制条件** (可选): 
  - `attr:属性名:最小值` - 需要最少属性值
  - `currency:货币名:最小值` - 需要最少货币数
  - 多个限制用逗号分隔

**示例**:
```
注册合成 ITEM_010*高级丹*ITEM_001:3,ITEM_002:2
注册合成 ITEM_015*魔法剑*ITEM_005:1,CUR_001:500*attr:魅力:50
注册合成 ITEM_020*终极装备*ITEM_010:5,ITEM_015:3,CUR_002:1000*attr:体力:100,currency:金币:50000
```

### 7.2 查看配方
```
配方 [过滤词]
```
玩家命令，查看所有配方或搜索特定配方。

### 7.3 执行合成
```
合成 产物代码
合成 配方编号
```
玩家命令，消耗材料进行合成。

## 八、二手市场

### 8.1 配置二手市场

**查看配置**:
```
二手设定 查看
```

**设置手续费** (百分比 2-5%):
```
二手设定 手续费:3
```
设置玩家转售时要扣除的手续费百分比。

**启用市场**:
```
二手设定 开启
```

**禁用市场**:
```
二手设定 关闭
```

### 8.2 物品二手设置

在注册物品时设置是否允许二手：
```
上载物品 物品名*描述*次数*效果*允许二手
```
- 最后一项为 `Y` 则允许二手销售
- 最后一项为 `N` 则禁止二手销售

**建议**:
- **允许二手**: 普通消耗品、装备等
- **禁止二手**: 一次性活动奖励、限定物品、账户绑定物品

### 8.3 玩家使用二手市场
```
上架二手 物品码 价格
```
玩家命令，将物品放上二手市场。

## 十、同步地点池子

### 10.1 自动同步
```
同步踩点池
```
将地点系统中的所有地点自动创建对应的抽取池。

**工作方式**:
- 地点名 → 地点名+池 (如 "城镇" → "城镇池")
- 已存在的池子会被跳过
- 不删除任何已有的池子

**应用场景**: 
- 地点系统启用后，一键创建踩点奖励池
- 之后可对每个池子单独配置物品

## 十一、配置管理最佳实践

### 初始化步骤

**第1步：创建属性**
```
我创建属性 体力 0 100 50
我创建属性 理智 0 100 80
我创建属性 魅力 -50 50 0
```

**第2步：创建货币**
```
注册货币 金币*通用货币
注册货币 钻石*高级货币
```

**第3步：注册物品**
```
上载物品 医疗包*恢复生命*-1*体力+30*Y
上载物品 魔法卷轴*增加魔法*0*理智+50*N
```

**第4步：创建池子**
```
注册池子 主池 fixed
上架池子 主池 ITEM_001*50
上架池子 主池 ITEM_002*30
```

**第5步：配置商城**
```
上架商城 ITEM_001*100金币
上架商城 ITEM_002*50钻石
```

### 日常维护

- **每周检查**: 
  - 物品列表是否完整
  - 池子配置是否平衡
  - 市场物品价格是否合理

- **问题调整**:
  - 物品过多/过少：修改池子权重
  - 经济崩溃：调整商城价格或物品产出
  - 玩家反馈：增删物品或调整属性效果

### 常见错误

| 错误 | 原因 | 解决方案 |
|------|------|--------|
| 物品无法使用 | 物品码错误或不存在 | 检查物品列表确认码值 |
| 合成失败 | 材料不足或不符合限制 | 检查玩家背包和属性值 |
| 抽取失败 | 次数用尽 | 检查每日限制配置 |
| 二手市场错误 | 物品禁止二手 | 查看物品是否允许二手销售 |
| 互动物品无法施加 | 对象不存在或编码错误 | 确认目标玩家和物品码 |

## 十二、物品系统编码表

| 前缀 | 说明 | 范围 | 示例 |
|------|------|------|------|
| `CUR_` | 货币 | CUR_001~999 | 金币、钻石、积分 |
| `ITEM_` | 普通物品 | ITEM_001~9999 | 装备、消耗品、任务物品 |
| `INTER_` | 互动物品 | INTER_001~999 | 治疗术、诅咒术 |
| `MARK_` | 标记 | MARK_001~999 | 成就、勋章、身份标记 |
| `SPEC_` | 特殊物品 | SPEC_001~999 | 测试用、隐藏物品 |

## 十三、存储位置
- **物品注册表**: `item_registry`
- **货币定义**: `item_currencies`
- **属性定义**: `rpg_attr_defs`
- **池子定义**: `pool_definitions`
- **合成配方**: `craft_recipes`
- **二手市场配置**: `market_system_config`

## 十四、权限要求
所有管理命令都需要 **管理员权限**。

## 十五、常见问题

**Q: 应该先做什么？**
A: 推荐顺序：属性 → 货币 → 物品 → 池子 → 商城 → 合成

**Q: 物品和互动物品有什么区别？**
A: 
- 物品：玩家对自己使用（使用方式 `普通使用`）
- 互动物品：玩家对他人使用（使用方式 `施加`）

**Q: 如何平衡物品产出？**
A: 
- 监测玩家背包和货币
- 调整池子概率权重
- 调整商城价格
- 增减物品产出源

**Q: 二手市场手续费应该设多少？**
A: 推荐 2-3% 防止炒作，不过高。

**Q: 能否删除已注册的物品/货币？**
A: 目前不支持删除，建议停用而非删除，以保持向后兼容。
